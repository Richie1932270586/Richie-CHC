# PRD｜Factory Exception Agent

## 背景

厂内物流、仓储和质检协同场景中，异常问题常常依赖 SOP、历史经验和多角色协同。对产品 demo 来说，最适合先收敛成一个单点流程，而不是一开始做复杂平台。

## 用户

- 仓储专员
- 线边物流人员
- 班组长
- 质检协同人员

## 场景

用户在收货、上架、发料、线边补料、质检协同过程中，遇到短缺、错发、不良品、批次异常、库位异常或停线风险，需要快速知道：

- 这是什么类型的问题
- 先做什么
- 查什么 SOP
- 是否要升级
- 是否要生成工单或通知草稿

## 核心问题

1. 信息散在 SOP、FAQ、经验里，不容易快速定位
2. 现场人员知道“有问题”，但不一定知道“下一步动作”
3. 一些敏感动作不能自动执行，但又需要系统帮忙整理草稿

## v1 范围

只做一个主流程：

输入异常描述 -> Agent 判定 -> 检索 + 查表 -> 输出建议 -> 生成草稿 -> 人工确认

## 非目标

- 不做多 Agent 编排
- 不做真实系统对接
- 不做权限体系
- 不做复杂流程引擎
- 不做训练式微调闭环

## 用户流程

1. 用户输入异常描述
2. Agent 识别问题类型
3. Agent 检索知识库并查询表格
4. Agent 返回结论、步骤、风险提醒、引用
5. Agent 推荐动作草稿
6. 命中敏感动作时要求人工确认

## 页面结构

- 对话输入区
- Agent 回复区
- 证据 / 引用区
- 建议动作区
- 人工确认区
- 模式切换区（RAG 开关、light/full、only-tool）

## 功能列表

- 异常分类
- 本地 RAG
- inventory / incidents / owners / template 查询
- 工单草稿生成
- 升级通知草稿生成
- 处理记录草稿生成
- 敏感动作人工确认

## 验收标准

- 没有 API Key 时也能启动并完成完整演示链路
- 至少覆盖 6 类异常类型识别
- 回答中包含结论、处理步骤、风险提醒、引用来源
- 页面能展示证据、动作草稿和人工确认
- 能通过脚本重建 demo 数据和知识库索引
- README 对非研发同学可读

## 后续迭代方向

1. 接入真实企业 IM / 工单系统
2. 增加更稳定的检索与 rerank
3. 增加结构化字段抽取
4. 增加审批流与确认日志
5. 引入更正式的评测与监控
