# 用户流

## 主流程

1. 用户输入异常描述
2. Agent 判断问题类型
3. Agent 决定是否检索知识库
4. Agent 决定是否查询本地表格
5. Agent 汇总证据和规则
6. Agent 输出结论、处理步骤、风险提醒、引用来源
7. Agent 生成可编辑草稿
8. 如果动作敏感，则进入人工确认

## 详细说明

### 输入异常

用户输入自由文本，例如：

- 零件短缺
- 标签和实物不一致
- 疑似不良品
- 批次重复异常

### Agent 判定

当前版本支持：

- 短缺
- 错发
- 不良品
- 批次异常
- 库位异常
- 高优先级停线风险

### 检索

如果不是 only-tool 模式，且 RAG 开启，Agent 会从本地知识库中召回 SOP / FAQ / 规则片段。

### 查询表格

Agent 会根据问题类型和文本中的实体，查询：

- inventory.csv
- incidents.csv
- owners.csv
- ticket_templates.json

### 输出建议

标准输出结构：

- 结论
- 处理步骤
- 风险提醒
- 引用来源

### 触发草稿动作

Agent 可生成：

- 工单草稿
- 升级通知草稿
- 处理记录草稿
- 责任确认草稿
- 对外通知草稿

### 人工确认

以下情况必须人工确认：

- 涉及报废
- 涉及停线风险升级
- 涉及责任归属判断
- 涉及对外通知
