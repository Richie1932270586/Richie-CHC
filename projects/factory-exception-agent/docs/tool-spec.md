# Tool / Skill 规格

## 1. `kb_retrieval`

- 名称：知识检索
- 输入：用户问题文本、问题类型、RAG 配置
- 输出：命中的 SOP / 规则 / FAQ 片段，附分数和来源
- 失败情况：知识库为空、未命中相关文档、索引未生成
- 是否敏感：否
- 是否需要人工确认：否

## 1.1 `office_folder_import`

- 名称：Office 文件夹导入
- 输入：本地文件夹路径、是否覆盖旧导入结果、是否递归扫描
- 输出：导入结果摘要、已生成的 markdown 文档、重建后的索引状态
- 失败情况：路径不存在、路径不可读、Office 文件损坏、提取不到文本
- 是否敏感：中
- 是否需要人工确认：否

## 2. `inventory_lookup`

- 名称：库存查询
- 输入：物料编码、物料名、批次号或用户原始描述
- 输出：库存余量、安全库存、库位、状态
- 失败情况：未提取到实体、表格中无匹配行
- 是否敏感：否
- 是否需要人工确认：否

## 3. `incident_lookup`

- 名称：历史异常台账查询
- 输入：问题类型、物料编码、批次号或用户原始描述
- 输出：历史异常记录、严重程度、处理结果
- 失败情况：没有历史匹配记录
- 是否敏感：否
- 是否需要人工确认：否

## 4. `owner_lookup`

- 名称：责任部门映射查询
- 输入：问题类型
- 输出：责任部门、责任人、升级联系人建议
- 失败情况：没有配置该问题类型的 owner
- 是否敏感：中
- 是否需要人工确认：如果输出被用于责任归属判断，则需要

## 5. `ticket_draft`

- 名称：工单草稿生成
- 输入：问题类型、风险等级、库存信息、历史异常、owner 建议
- 输出：可编辑工单草稿
- 失败情况：模板缺失、上下文不完整
- 是否敏感：低
- 是否需要人工确认：通常不需要

## 6. `handling_record`

- 名称：处理记录草稿生成
- 输入：异常描述、当前建议、引用和 owner 建议
- 输出：处理记录草稿
- 失败情况：模板缺失
- 是否敏感：低
- 是否需要人工确认：通常不需要

## 7. `quality_escalation`

- 名称：升级通知草稿生成
- 输入：问题类型、风险等级、影响范围、引用来源
- 输出：给班组长 / 质检的升级通知草稿
- 失败情况：上下文不足
- 是否敏感：高
- 是否需要人工确认：是

## 8. `owner_assignment`

- 名称：责任确认草稿生成
- 输入：问题描述、owner 查询结果、历史台账
- 输出：责任确认草稿
- 失败情况：owner 查询为空
- 是否敏感：高
- 是否需要人工确认：是

## 9. `external_notification`

- 名称：对外通知草稿生成
- 输入：异常描述、影响范围、建议口径
- 输出：供应商 / 外部协同通知草稿
- 失败情况：上下文不完整
- 是否敏感：高
- 是否需要人工确认：是
